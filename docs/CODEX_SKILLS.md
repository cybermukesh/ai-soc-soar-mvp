# Codex Skills For This MVP

The repo includes project-local skills under `codex-skills/`. They are operating rules for fast development during the hackathon.

## Skill Map

| Skill | Use |
| --- | --- |
| `ai-soc-architecture` | Architecture boundaries, SIEM-agnostic design, SOAR approval rules. |
| `wazuh-connector` | Wazuh/OpenSearch ingestion, field mapping, fixtures, connector tests. |
| `llm-triage-agent` | Prompt design, JSON verdicts, evidence, confidence, prompt-injection defense. |
| `soar-integration` | n8n, Shuffle, Slack, webhook payloads, approval-gated response actions. |
| `secure-soc-mvp` | Secrets, audit logs, untrusted alert content, tenant-ready controls. |

## Development Rule

Every new feature should map to one of these skill areas. If it does not, it is probably outside the 7-day MVP scope.
