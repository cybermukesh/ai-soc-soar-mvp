Day 4

AI Triage + SOC Case Lifecycle

In Progress

Day 4 is focused on making triage and case handling useful for L1/L2 analysts: deeper evidence, investigation steps, review feedback, case ownership, SLA fields, escalation, and resolution tracking.

AI Triage Completed

  • Structured triage decisions with verdict, confidence, risk score, summary, evidence, MITRE context, and SOAR recommendation.
  • Expanded triage output with impacted entities, L2 investigation steps, containment steps, resolution criteria, analyst questions, and recommended actions.
  • Cache-aware triage flow to avoid repeated low-value model calls for duplicate alert signatures.
  • Noise-reduction engine now calculates signal score, noise score, analyst priority, queue, suppression decision, suppression reason, and correlation key.
  • Verified current live dataset: 100 alerts reduced to 38 analyst work items, an estimated 62 percent workload reduction.
  • Force-refresh option preserved for analyst-controlled retriage.
  • Triage history endpoint stores analyst disposition and notes.
  • Frontend AI Triage Workbench shows raw event access, evidence, investigation steps, containment, resolution criteria, analyst questions, and review feedback.

Case Management Completed

  • Incident and incident event schemas added for persistent case tracking.
  • Case APIs support create, list, status update, timeline retrieval, and event creation.
  • Case board added with SOC phases: New, Triage, Investigation, Response, Closed.
  • Alert-to-case handoff added from AI Triage through the Raise Ticket action.
  • Case fields expanded with owner, ticket reference, priority, SLA due time, escalation flag, close reason, and resolution summary.
  • Timeline events track creation, status change, notes, investigation, containment, recovery, and ticket events.

UI and Analyst Workflow Completed

  • Wazuh-style responsive dashboard polish with compact navigation, clean panels, and mobile/tablet layout rules.
  • Duplicate React key issue fixed for repeated Wazuh alert IDs.
  • Executive dashboard now shows ingestion state and avoids misleading zero-alert messaging while persisted counts are available.
  • Case board cards show priority, severity, risk, ticket, owner, SLA, escalation, summary, and resolution notes.
  • Role-aware actions continue to restrict write operations to admin/analyst users.

Control Plane Completed

  • Persistent AI provider settings added for OpenAI, Anthropic, Ollama, and offline heuristic mode.
  • Persistent threat-intel provider settings added for VirusTotal, AbuseIPDB, OTX, MISP, and local IOC matching.
  • Secrets are masked in API responses and stored through the backend secret helper.
  • Admin-only settings APIs and frontend AI & Intel page added.
  • Provider health checks report configuration state without exposing keys.
  • Alert flood handling started with paginated and filtered persisted-alert search.

Still In Progress

  • Strict confidence threshold routing and analyst escalation policy.
  • Triage service wiring to active cloud/local model settings.
  • Threat-intel IOC lookup, cache, and enrichment details inside triage.
  • Incident grouping and deduplication as first-class persisted objects.
  • Evidence attachments and richer investigation notebook.
  • SOAR playbook dispatch to Shuffle with human approval gates; n8n remains optional.
