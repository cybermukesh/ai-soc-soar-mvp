Day 6

SOAR Execution and Analyst Control

In Progress

Day 6 now has the first automation backbone: n8n connector state, workflow templates, trigger API, run logs, RBAC, and frontend request flow.

Completed This Sprint

  • GET /api/v1/automation/connectors lists automation connectors and n8n configuration state.
  • GET /api/v1/automation/workflow-templates exposes the first n8n test webhook template.
  • POST /api/v1/automation/workflow-templates/n8n-test-webhook/trigger dispatches bounded case and alert context to n8n.
  • GET /api/v1/automation/workflow-runs returns persisted workflow execution history.
  • Automation write path is limited to admin and analyst roles and creates audit log records.
  • Frontend SOAR console lets analysts select workflow, case, alert, and approval note before requesting a run.
  • Analyst-requested containment workflows are recorded as pending_approval instead of dispatching immediately.

Current Integration Gate

  • Set N8N_WEBHOOK_URL in the backend environment after n8n is installed.
  • Restart the backend so connector state changes from not_configured to enabled.
  • Run the trigger smoke test and confirm the run appears in workflow history.

Still Building

  • Shuffle adapter and SOC-native workflow template pack.
  • Admin approve/reject controls for pending destructive actions.
  • Outbound request signing, retry/backoff, and dead-letter tracking.
  • Case timeline integration for queued, running, success, and failed automation steps.
