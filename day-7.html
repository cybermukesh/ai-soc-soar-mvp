Demo Flow

  1. Login with the seeded admin user.
  2. Show executive metrics: raw alerts, analyst items, grouped duplicates, solved/unsolved cases.
  3. Open AI Triage and show alert evidence, raw event, L2 steps, containment, and resolution criteria.
  4. Open Correlation Groups and explain how duplicate/noisy alerts are compressed.
  5. Add a local IOC and enrich an alert without sending lab data to external providers.
  6. Create or update a case, add investigation history, then show the lifecycle board.
  7. Request a SOAR action and show pending approval before admin approval/rejection.
  8. Open Admin and demonstrate role assignment, revocation, approval, and audit records.

n8n Setup Needed

Install n8n on the lab host when live SOAR dispatch is required. The MVP already stores connector state and run history; it only needs the webhook URL.

docker volume create n8n_data
docker run -d --name n8n --restart unless-stopped \
  -p 5678:5678 \
  -e N8N_HOST=0.0.0.0 \
  -e N8N_PORT=5678 \
  -e N8N_PROTOCOL=http \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n:latest

Then set N8N_WEBHOOK_URL=http://<n8n-host>:5678/webhook/<workflow-path> in the backend environment and restart the backend.
