Judge Summary

What We Are Building

The product is an AI-assisted SOC/SOAR layer for analysts. It reads SIEM alerts, removes repeated noise, gives concise attack analysis, groups related events into incidents, and starts response actions through n8n or Shuffle after analyst approval.

Industry Pain

Why It Matters

SOC teams face alert overload, repeated false positives, limited analyst time, and expensive enterprise automation platforms.

High-Level Architecture

From SIEM Alerts To Analyst Action

1Wazuh / OpenSearch

Read alerts through API and index queries.

2FastAPI

Ingestion, connector endpoints, and audit APIs.

3Normalized Schema

Vendor-neutral fields for future SIEM replacement.

4AI Triage

Low-cost LLM returns strict JSON verdicts and evidence.

5Incidents

6SOAR

Approval-gated n8n or Shuffle workflows.

7Dashboard

Analyst console for alerts, incidents, and actions.

Token And Cost Controls

Low-token Cheap Cloud Policy

Do not fine-tune in MVP

Use prompt engineering plus structured JSON.

Send compact context

Only normalized alert fields and compact history go to the model.

Cache duplicate triage

Repeated alerts reuse prior verdicts unless evidence changes.
